The commons underneath

Roost as a protocol.

Behind the FallHub-hospitality showroom, Roost also runs as a host-owned protocol — a directory of stays declared by hosts themselves, not owned by any middleman. Every FallHub instance can publish its availability to the commons via a JSON manifest. Guests browse the commons directly.

The commons is optional. You can run FallHub without ever listing on Roost. But if you do list, you get a distribution channel that takes zero fees, harvests zero data, and can never delist you for arbitrary reasons.

  • Zero-fee listing · zero booking commission
  • Zero data harvest · guest contact goes direct
  • Protocol-based · anyone can host their own Roost node
  • JSON registry · queryable, forkable, mirrorable
